# Record Storage API Server

Простой HTTP сервер для хранения и получения записей. Сервер сохраняет данные между запусками в JSON файл, не использует
внешних баз данных и работает только на стандартной библиотеке Python.

## Возможности

- Сохранение записи (только поле `recording`)
- Получение всех записей
- Получение записи по уникальному ID (автоинкремент)
- Сохранение данных между запусками
- Валидация входных данных
- Логирование всех операций

## Установка и запуск

### Требования

- Python 3.8+
- Никаких внешних зависимостей не требуется

### Запуск сервера

```bash
python main.py
```

## Сервер запустится на порту 8000

- Endpoints:
  POST /create
  GET /get_list
  GET /get_detail/{id}

## Остановка сервера

- Нажмите Ctrl+C

## Создание записи

- путь http://localhost:8000/create
- Метод запроса POST
- Тело запроса {"recording": "Текст записи"}
- Ответ Успешный ответ (201 Created)
- Ошибка (400 Bad Request):

## Получение всех записей

- путь http://localhost:8000/get_list
- метод GET
- Успешный ответ (200 OK)
- Ошибка (400 Bad Request)
- тело ответа {
- "1": {
-        "recording": "Первая запись"
- },
- "2": {
-        "recording": "Вторая запись"
- }
- }

## Получение записи по ID

- путь http://localhost:8000/get_detail/{id}
- метод GET
- Успешный ответ (200 OK)
- Ошибка (400 Bad Request)
- {
- "1": {
-        "recording": "Первая запись"
- }
- }

## Валидация данных

Сервер принимает только записи с полем recording:

- ✅ {"recording": "Hello world"} - валидно
- ❌ {"description": "test"} - нет поля recording
- ❌ {"recording": ""} - пустая строка
- ❌ {"recording": " "} - только пробелы
- ❌ {"recording": 123} - не строка
- ❌ {"recording": "A"*1001} - длиннее 1000 символов
